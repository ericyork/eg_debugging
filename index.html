Source of Bugs

All bugs come from one of two sources. They are either syntax erorrs or logic erorrs. Syntax errors are when a spelling or grammar error, i.e. a typo, prevents the browser from correctly reading the code. A logic error is when the code is correct, but doesn't make sense or doesn't do what you want it to. Always check for syntax errors first, as typing mistakes are most common.

The most common sources of syntax errors are:

  1. missing punctuation, such as angle brackets or semi-colons
  2. missing end-tags or improperly nested tags
  3. misspelled names of classes, tags, or ids.

Best Defense

The best defense against bugs is to prevent them. Move through tasks one step at a time, checking your work as you go. If you make a change, check in the browser to see what effect it had. If something breaks, look back at what you just did. What you don't want is to write hundred lines of code and then notice that the page is broken: how would you ever know where the errors are? So your first secret weapon is to only make one change at a time and then to test it.

You other secret weapon is to use the browser's inspector to view the element and inspect what is going on.

If an included asset, like an image or CSS file, isn't working, check the file path to make sure it's correct.

Validate Your Code

Inevitably you will make errors and not notice that you have made them or not be able to find them. When you do, you can turn to code validators. The W3C maintains excellent tools that can scan your code for common errors.

Once you've pasted in and checked your code, you should work your way through any errors you get. Of course, many of the errors you get will be mysterious at first, so google them to learn how to resolve them.

If you get no errors, that's a good sign it's an error of logic (lack of understanding) rather than and error of syntax (a typo or misspelling).

Add a Border

One way to debug logical CSS errors is to add a visible border around an element. For example you can write a rule like .debug {border: 1px solid red;} and then add that class to whatever elements you need. This will show where elements are actually positioned and the space they are actually taking up.

Look it up on Google

Sometimes you won't be able to find the error. In that case, you should look up and read about the property or element you're trying to use. Find a working example and try to implement that, first, and then modify it. Learning how to effectively search for answers is a big part of learning to write code.

Isolate the Problem

Figuring out where, roughly, the problem is occuring, is half the battle. If you can rule out entire files or sections of your code, then you can focus your attention on what's left.

Try removing sections of the code systematically to see if the errors persists. If you remove code and the error disappears, that's a sign the error was in the code you removed.

Go back to when it was working and re-add the code a bit at a time until the error appears again. What did you just add?

Replace parts of the code with a known working example and start modifying it a little bit at a time, checking as you go.
